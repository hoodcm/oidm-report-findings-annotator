The AI will make mistakes — review every finding it imports.

AI tools sometimes mix up which report a finding came from, add details the report doesn't say, or skip findings in longer sentences. The prompt below reduces these errors but can't catch all of them.

Before you paste or attach anything: report text leaves your machine when you send it to an AI tool. Make sure your reports are de-identified first, per your institution's policy.

Run an LLM to extract findings

Use the AI chat tool of your choice to pre-extract structured findings from your radiology reports. This page gives you the prompt to use, shows you what to attach, and explains how to fix problems if something doesn't import.

Every finding object must include these four fields: record_id · finding_name · presence · source_text

Everything else is optional. See the JSON field reference below for the full spec.

      Why is the prompt so long?
      

        The prompt includes your full finding list so the AI uses your finding
        names from the start, instead of inventing its own wording that has to be
        cleaned up later. Don't worry about the length: even with several hundred
        findings, this prompt is roughly a 10–20 page document, and today's AI
        tools reliably read hundreds of pages in one message. Pasting it works
        better than attaching it as a file — pasted text is always fully in the
        AI's view, while attached files are sometimes only skimmed or searched.
        Expect a strong starting point, not perfection: you'll review every finding
        in the annotator.
      

    

  


  
  

    
Step 2 — Run the prompt against your reports

    

      Paste the prompt into your AI chat, then attach the same reports CSV you plan to load into the annotator, in the same message. Attaching the same file the annotator already knows about is what lets the AI copy record IDs correctly.
    

    

      If your reports file is large, the AI may reply in a few messages instead of one — that's expected and the annotator's import handles it fine; just repeat Step 3 for each reply.
    

    

      
Can't attach a file in your chat?

      
Paste your reports directly into the chat instead, in labeled batches — write Report <record_id>: before each report's text. The same de-identification reminder above still applies.

    

    

      The reports CSV format is documented at the Reports CSV Format Guide.
    

  


  
  

    
Step 3 — Import the AI's reply into the annotator

    

      Open the annotator, open Import & add data in the sidebar, and either:
    

    
    
      
  • Paste the reply directly — use the "Or paste the AI's reply instead" box. This is the simplest path: no file to save, and nothing for a word processor to mangle.
    • 
      
  • Save it as a file first, if you'd rather keep a copy — use a plain-text editor (not a word processor's default rich-text mode, which can silently swap in curly "smart quotes" that break the file) or your AI tool's own "download" option when it has one. Then drop the file on the same import area.

    If something doesn't import
    

      The annotator validates every row before import and shows you what to fix. Common cases:
    

    

      

        
          

            Error
            Cause
            Fix
          

        
        
          

            missing required field
            A row lacks record_id, finding_name, presence, or source_text.
            Every row must include all four. Update your extraction.
          

          

            record_id not in loaded reports
            The row references a record that wasn't imported as a report.
            Re-import the reports CSV with this record included, or check for typos.
          

          

            source_text not in named report
            The text doesn't appear in the report's FINDINGS section. Often a sign of cross-attribution (the AI batched multiple reports with the wrong record_id) or paraphrase (it didn't quote verbatim).
            Check the validation panel — if the source_text matches another loaded report, your record_id is mismatched. If a close match was found in the right report instead, click Use this sentence to fix it in one click.
          

          

            source_text matches multiple sentences
            The same body text appears under multiple section headers (templated report).
            Add the section header to source_text. Example: "Celiac artery: no dissection."
          

          

            presence value not recognized
            A value other than the ones listed in the field reference below.
            Update the extraction to emit one of the allowed values.
          

          

            attribute value not recognized
            A canonical enum attribute (laterality, severity, etc.) has a value outside its vocabulary.
            Use the allowed values listed in the field reference below.
          

        
      

    

    

      Still have rejected rows after fixing what you can? The validation panel's Copy fix-it prompt for the rejected rows button packages everything a fresh AI chat needs — the rejected rows, their errors, and the report text — so you can paste it into a new chat and get corrected rows back. Re-importing the reply adds to what you already have; it won't wipe out the findings that already imported cleanly.
    

    

      The validation panel also offers a Download error report (CSV) button that exports every rejected row with a _validation_error column appended, if you'd rather work from a spreadsheet.
